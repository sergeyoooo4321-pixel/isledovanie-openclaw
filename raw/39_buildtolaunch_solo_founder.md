## Что использует

### Контент и управление знаниями
- Сканирует newsletter-фиды дважды в день, ищет пересечения со своей работой
- Индексирует статьи и ведёт поисковую базу памяти
- Выделяет пересечения входящего контента с её опытом

### Автоматизация workflow
- Обработка inbox и черновики комментариев
- Публикация в соцсети и синдикация контента
- Анализ Search Console и аудиты сайтов
- Синхронизация блога между платформами
- Мониторинг состояния систем

### Оперативная работа
- Менеджмент БД
- Построение сайтов с бизнес-партнёром

## Что работает хорошо

**Автономное построение контекста:**
> "The inbox cron scans my newsletter feed, picks up new posts, reads each one, and searches my memory database for related experience" — убирает ментальную нагрузку поиска углов для комментариев

**Командная коллаборация:**
Партнёр принял инструмент самостоятельно через Telegram без инструкций — использует для обновлений сайта. Интерфейс ощущается "less like a developer tool and more like texting a coworker."

**Ускорение workflow:**
Задачи с необходимостью суждений теперь имеют чёткие точки входа, а не чистый лист.

---

## Что не работает / что сломалось

**Катастрофа с базой данных:**
При запросе обновить приложение для изучения языков через Telegram — агент стёр все таблицы БД при отладке, уничтожив продакшн-данные.

**Пробелы в интеграции:**
Передачи между запланированными заданиями остаются ручными. Соединить вывод одного задания как триггер для другого — требует дополнительной работы.

**Сложность настройки:**
Первоначальная конфигурация заняла дольше, чем следует из документации; security defaults потребовали ручного усиления перед продакшн.

- Bind to `127.0.0.1` only
- Zero third-party skills from ClawHub

## ROI Framework: когда OpenClaw выигрывает

**Когда стоит использовать OpenClaw:**
- Задачи, требующие AI-суждения и адаптации
- Работа, которая меняется по ходу выполнения
- Высокоценные активности, которые ты пропускаешь из-за контекстных накладных расходов

**Когда лучше альтернативы:**
- Фиксированные, повторяемые workflows → n8n или cron jobs
- Интерактивная разработка → Claude Code
- Одноразовые вопросы → ChatGPT/Claude

**Реальные затраты:**
Инфраструктура может быть бесплатной (Oracle Cloud, Supabase); расходы идут от API-вызовов AI-модели по частоте и объёму.
